# Настройка QUIK для TransToQuik

## 1. Установка терминала

1. Скачайте QUIK с [сайта поддержки АРКА](https://arqatech.com/ru/support/files/).
2. Для обучения можно запросить демо-доступ: [форма заявки](https://arqatech.com/ru/support/demo/).
3. Установите терминал и запомните путь к каталогу (например `C:\QUIK Junior\`).

## 2. Нативная библиотека

Скопируйте `trans2quik.dll` из каталога QUIK в одно из мест:

| Сценарий | Куда копировать |
|----------|-----------------|
| Пример из репозитория | `libs/trans2quik.dll` → при `dotnet build` попадает в `bin` рядом с `.exe` |
| Ваше приложение | рядом с `.exe` (папка `bin` или каталог публикации) |
| CI / тесты без QUIK | не требуется (unit-тесты используют моки) |

Приложение **не копирует** DLL во время выполнения — только при сборке (из `libs/`) или вручную.

> Файл `trans2quik.dll` **не входит** в NuGet-пакет `TransToQuik` из соображений лицензирования.

## 3. Настройки терминала

В QUIK обязательно:

1. **Сервис → Настройки → Торговля → Внешние транзакции** — включить.
2. Запускать QUIK **не от имени администратора**, если ваше приложение работает без повышенных прав (или наоборот — права должны совпадать).
3. Терминал должен быть **запущен и подключён** к серверу до вызова `Connect`.

## 4. Где взять торговый счёт (`Account`)

В транзакциях TransToQuik поле **`Account`** — это параметр `ACCOUNT` в TRANS2QUIK: **торговый (депозитарный) счёт** для выставления заявки на выбранном рынке.

Это **не** то же самое, что:

| Поле | Пример | Где используется |
|------|--------|------------------|
| Логин QUIK | `U0222262` | Вход в терминал |
| Код клиента | `10434` | Идентификатор на демо-стенде |
| `Account` в заявке | `NL0011100043` | Параметр транзакции `ACCOUNT` |

### Как посмотреть в QUIK

1. **Торговля → Заявка** — выберите класс (`ClassCode`, например `QJSIM`) и инструмент (`SecCode`, например `VTBR`). В форме заявки отображается счёт, который подставится в транзакцию — его и указывайте в `Account`.
2. **Счета / Лимиты / Портфель** — список счетов, доступных по нужному рынку.
3. **Таблица заявок** — откройте свою заявку по этому инструменту и посмотрите поле счёта в свойствах строки.

### Разные рынки — разные счета

Счёт привязан к **рынку и классу инструмента**. Для фондового демо (`QJSIM`) и для срочного (`SPBFUT…`) значения `Account` обычно **разные**. Перед отправкой транзакции проверяйте счёт именно для того `ClassCode`/`SecCode`, с которым работаете.

В примере значение задаётся в `appsettings.Development.json` → `Quik:Account`.

## 5. Типичные проблемы

| Симптом | Что проверить |
|---------|----------------|
| `DllNotConnected` | QUIK не запущен или путь к каталогу неверный |
| `DllVersionNotSupported` | Версия `trans2quik.dll` не совпадает с терминалом |
| Транзакции не проходят | Внешние транзакции отключены в настройках |
| Нет callback заявок/сделок | Подписка `SubscribeOrders` / `SubscribeTrades` и `StartOrders` / `StartTrades` |

## 6. Снятие заявок через Trans2QUIK

### Не работает в `trans2quik.dll`

Следующие `ACTION` есть в справочнике QUIK для импорта из **tri-файла**, но **не поддерживаются библиотекой Trans2QUIK** (см. [руководство QUIK, импорт через API](https://euvgub.github.io/quik_user_manual/ch6_10.html)):

| `ACTION` | Заявлено в QUIK | Через Trans2QUIK |
|----------|-----------------|------------------|
| `KILL_ALL_ORDERS` | снять все заявки | **нет** |
| `KILL_ALL_STOP_ORDERS` | снять все стоп-заявки | **нет** |
| `KILL_ALL_NEG_DEALS` | внебиржевые и РЕПО | **нет** |

В enum `QuikAction` эти значения есть для совместимости со строкой транзакции, но QUIK их **не примет** через DLL.

### Что использовать вместо

| Задача | `ACTION` | В TransToQuik |
|--------|----------|---------------|
| Снять одну заявку | `KILL_ORDER` + `ORDER_KEY` | `QuikKillOrderTransaction` |
| Снять стоп-заявку | `KILL_STOP_ORDER` + `STOP_ORDER_KEY` | `QuikKillStopOrderTransaction` |
| Снять внебиржевую / РЕПО | `KILL_NEG_DEAL` + `ORDER_KEY` | `QuikKillNegDealTransaction` |
| Снять все свои заявки | цикл по активным заявкам | подписка `SubscribeOrders` → для каждой `KILL_ORDER` |

Пример в репозитории: `TransQuik.Example` снимает одну заявку через `QuikKillOrderTransaction`.

## 7. Демо-режим

На учебных серверах Московской биржи торги моделируются по историческим данным. Уточняйте актуальные классы кодов (`ClassCode`), инструменты (`SecCode`) и счета в документации вашего брокера / демо-стенда.
